# Story Bible, Relasi & Faksi

## Story Bible — `src/lib/storyBible.ts` adalah sumber tunggal
Sumber tunggal opsi (genre/tone/POV/pacing) sekaligus pemformatannya untuk AI. Nilai disimpan sebagai slug/JSON di Dexie, tapi jangan pernah kirim slug mentah ke AI — pakai `formatBibleBlock`/`formatBibleRuleLine` (slug→label, buang baris kosong, cap per-key via `BIBLE_AI_MAX_CHARS`). Jangan duplikasi tabel label di komponen. **Panel** = `src/features/lore/components/BiblePanel.tsx` + `useBiblePanel.ts` (autosave debounce 800ms + flush-on-blur; `useBiblePanel` menyinkronkan ulang dari DB saat perubahan eksternal — restore/impor/asisten — HANYA bila tak ada edit tertunda, dijaga penanda `autoSaveTimerRef`, jadi jangan clear timer tanpa me-null-kan ref). **Mode konteks `minimal`** (`buildContextBlock` di `services/ai/index.ts`) sengaja menyertakan Premis+Genre+Tone+POV+Tema dan MEMBUANG Judul (Judul nyaris tak mengarahkan generasi).

## AI-assist Story Bible
`suggestBibleField(field, ctx)` (`services/ai/index.ts`) + prompt `AI_PROMPTS.BIBLE_ASSIST`/`BIBLE_ASSIST_FIELD_GUIDE` (`aiPrompts.ts`) + hook `useBibleAssist.ts` + komponen `AiAssistZone` (di `BiblePanel.tsx`). Menghasilkan draf SATU bidang (`tagline|premise|setting|themes|targetAudience`) yang konsisten dengan profil terisi. Profil disusun via `formatBibleBlock` (reuse — slug→label otomatis), **bidang target dikecualikan** agar model tak menyalinnya. Kreatif ter-grounded (temp 0.8), `actionType: 'expand'` (model penuh, bukan light), dibatalkan lewat `cancelAI('bible-assist')`; output dibersihkan dari fence/kutipan (`stripWrappers`). Hook = satu-saran-aktif dengan guard `reqId` (abaikan respons basi). UI cap panjang saat MENERAPKAN (`BIBLE_FIELD_CAPS`, selaras cap onChange). **AI tak pernah menulis DB** — pengguna memilih Ganti/Tambah lalu `flushField` menyimpan. `BIBLE_ASSIST_FIELD_GUIDE` adalah sumber tunggal panduan per-bidang; saat menambah bidang assist, perbarui map itu + `BIBLE_FIELD_TO_KEY`.

## Panel Relasi Karakter (`viewMode 'relationships'`)
Master-detail (daftar karakter kiri, profil+hubungan kanan; `RelationshipMapper.tsx` + `useRelationshipMapper.ts`). Tipe & label hubungan dari sumber tunggal `src/features/codex/relationshipTypes.ts` (`RELATIONSHIP_TYPES` + `getRelationshipLabel` sadar-arah) — sama seperti `CodexDetailModal`; **jangan** definisikan daftar tipe lokal lagi (metadata visual per-tipe boleh di komponen, di-key oleh `value`). Daftar kiri terurut per jumlah koneksi + filter "hanya yang terhubung"; diagram ikatan hanya tampil saat karakter punya ≥2 koneksi. Relasi disimpan di tabel `relationships` (project-scoped, sudah di `importRemap`/`deleteProject`/`backup`). *(Panel Relasi sengaja pakai SVG radial kustom, bukan force-graph — lihat jebakan force-graph di `codex.md`.)*

## Panel & Papan Faksi (#15, `viewMode 'factions'`)
Grup entitas + politik antar-faksi, NOL tabel baru. Faksi = entri Codex ber-**`factionTag`** (field inert, non-kosong ⇒ entri ini faksi; v29). Anggota faksi F = entri Codex yang `tags`-nya memuat `F.factionTag` (turunan tag, tanpa tabel keanggotaan). **Sumber tunggal logika = `src/lib/factions.ts` (murni, teruji ~17 tes):** `buildFactions` (entri→`Faction{id,name,tag,hidden,memberIds,board}`), `aggregateFactionRelations` (per pasangan faksi tak-berarah: `declared` relasi antar dua ENTRI faksi + `derived` tally relasi antar-ANGGOTA + `internal` kohesi; dedup multi-membership via `pairKey`), `factionPairSentiment` (deklarasi menang, else turunan dominan — **tanpa alarm/kontradiksi**; sekutu yang anggotanya bermusuhan itu SAH). **Filosofi "penulis mendeklarasikan, alat membukukan" + dua sinyal netral berdampingan.** `factionTag` = metadata alur kerja → **v1 TIDAK masuk KB AI/ekspor**. Hook `useFactions.ts`; `TYPE_STYLE`/`styleOf` (warna per tipe relasi) dipakai bersama dari `src/features/lore/relationshipStyles.tsx`. UI v1 = master-detail (toggle **"Daftar"**).

### Papan Faksi (fase 2, toggle "Kanvas")
Kanvas drag-and-drop React Flow, di-`React.lazy` (bundle + CSS `@xyflow/react` hanya dimuat saat mode Kanvas). Prinsip wajib **"logika murni = sumber kebenaran, React Flow = cermin"**: node/edge dirakit dari **`buildFactionBoard(factions, relData)`** (murni, di `factions.ts`; 1 node/faksi + posisi seed dari field inert **`CodexEntry.factionBoard?:{x,y}`** [v30] atau `fallbackSlot` grid deterministik; satu edge/pasangan via `factionPairSentiment`, `kind` declared/derived). Komponen: `faction-board/FactionBoardCanvas.tsx` (default export lazy) + `FactionCardNode.tsx` (custom node) + `FloatingEdge.tsx` (garis center-to-center) + `FactionInspector.tsx` (drawer kanan). **Posisi = kebenaran DB; React Flow pegang posisi transien saat drag, commit balik lewat `onNodeDragStop` → `db.codex.update({factionBoard})`.** Reconcile: node dikontrol via `useNodesState`, effect rebuild HANYA saat data/UI berubah (deps stabil → TAK jalan saat drag → mulus tanpa lompatan); posisi dipertahankan dari node yang sudah ada (kebenaran sesi), node baru di-seed dari `boardView.position`. Interaksi: drag+persist, **drag-to-connect** (handle connectable + `ConnectionMode.Loose` → `onConnect` → `ConnectDialog` pilih tipe → `addRelationBetween` tulis `relationships`), **inspector drawer** (klik kartu → sorot ego-network + panel anggota/relasi/kohesi/tetapkan-relasi), ciut/buka, snap-to-grid, "Susun rapi" (re-tile `fallbackSlot` + `fitView`), pan/zoom/minimap/controls bawaan. **`factionTag`/`factionBoard` inert menumpang objek codex → backup/impor/`deleteProject` TAK berubah** (pola `tension`/`worldStatus`/`namePalette`/`hidden`); relasi & keanggotaan menumpang `relationships`+`tags` yang sudah terdaftar di semua jalur. **Saat menambah tampilan/analitik faksi baru, turunkan dari `factions.ts` (matriks faksi×faksi = `buildFactionMatrix` belum dibuat) — jangan tambah tabel/FK; React Flow hanya lapisan render.**

> **⚠️ JEBAKAN warna edge (jangan diulang):** warna garis WAJIB lewat **`style` inline** di `<path>` FloatingEdge, BUKAN atribut `stroke` SVG — class `react-flow__edge-path` membawa CSS bawaan React Flow yang **menimpa** presentation attribute (CSS > atribut), jadi semua garis akan seragam abu-abu bila warna ditaruh di atribut `stroke`. *(Berbeda dari force-graph #14 yang hit-test-nya bermasalah; React Flow node = DOM biasa → klik/hover native, tak perlu deteksi koordinat manual.)*
