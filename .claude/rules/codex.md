# Codex: kategori, field, graf, lokakarya

**Meta-pola (berlaku untuk hampir semua fitur di file ini):** field opsional yang menumpang objek `CodexEntry`/`Chapter` (bukan tabel/FK baru) → **backup/impor/`deleteProject` tak berubah**. Skema Dexie diperluas dengan blok `version(N)` no-op append-only (field tak diindeks). Ini pola `namePalette`/`hidden`/`secret`/`worldStatus`/`factionTag`.

## Saran Entitas / orphan
`src/lib/orphanEntities.ts` — deteksi nama-diri yang sering muncul tapi belum di Codex (kebalikan Aho-Corasick); enrichment opsional via `enrichEntities`. Berjalan on-demand di main thread (`OrphanScanPanel`).

## Kategori Codex — `src/lib/codexCategories.ts` adalah sumber tunggal
Kategori bawaan (6, immutable) + registry ikon/warna yang boleh dipakai kategori KUSTOM. Kategori kustom per-proyek disimpan di tabel Dexie `codexCategories` (slug bebas → nilai `CodexEntry.category`; `CodexCategory` kini `string`). Baca via hook `useCodexCategories(projectId)`, resolusi label/ikon lewat `getCategoryLabel`/`getCategoryDef`/`CategoryIcon` — **jangan hardcode `<option>` kategori atau peta label lagi** di komponen. CRUD ada di `CategoryManagerModal`; menghapus kategori mengalihkan entri terkait ke `other`. Prompt AI enrich/expand tetap dibatasi ke 6 kategori bawaan (kustom bersifat manual). Tabel ikut backup (`backupService` v3).

## Template field per kategori (#17)
Dua field JSON inert + logika murni `src/lib/codexFields.ts` (+ tes), BUKAN tabel baru. Skema field hidup di `CustomCategory.fields?: CategoryFieldDef[]` (`{key,label,type,options?}`, tipe `text|textarea|number|select`); NILAI-nya di `CodexEntry.customFields?: CustomFieldValue[]` sebagai array **TER-DENORMALISASI** (`{key,label,value}`). Skema Dexie **v26** no-op append-only. **Denormalisasi label saat simpan** membuat SEMUA jalur baca self-contained → tak perlu meneruskan kategori ke jalur AI (jaga cache prompt byte-identik). Alur: `CategoryDef.fields` diteruskan lewat `mergeCategories`; editor skema di `CategoryManagerModal` (expandable, hanya kategori KUSTOM); `CodexForm` render seksi "Detail {kategori}" dari `fieldDefs` kategori aktif & bangun nilai via `buildFieldValues(defs, values)` di `useCodexPanel.handleSaveEntry`; render nilai via `resolveFieldValues` di `CodexDetailModal` (definition-list) & `CodexCard` (≤2 field). **KB AI & ekspor menyertakan field** (semua field, tanpa toggle per-field di v1): `formatFieldsForAI` ditempel di `buildCachedContextSegments`+`buildContextBlock` (`services/ai/index.ts`), `formatFieldsForMarkdown` di `codexToMarkdown`. **Konsekuensi disengaja:** ganti `label` tak mengubah entri lama sampai disimpan ulang; pindah kategori lalu simpan membuang nilai kategori lama. **Saat memperluas: tambah tipe field = perbarui render di `CodexForm` + editor di `CategoryManagerModal`; toggle privat per-field (includeInAI/Export) = denormalisasi flag ke `CustomFieldValue`.** Lihat #17 di `RENCANA-FITUR-WORLDBUILDING.md`.

## Lapis "Kebenaran Tersembunyi" (kanon vs rahasia penulis)
Dua field opsional di `CodexEntry`, BUKAN tabel baru. `hidden?: boolean` (entri sepenuhnya rahasia / mystery-box) & `secret?: string` (kebenaran penulis pada entri yang tetap publik, pola bible "Dunia percaya… / Yang sebenarnya…"). Skema v22 append-only. **Aturan inti — permukaan pembaca menyaring, konsistensi tidak:** entri `hidden` & isi `secret` **tetap** diumpankan ke KB AI (`buildCachedContextSegments` + jalur RAG `buildContextBlock` di `services/ai/index.ts` — `secret` ditempel sebagai baris `[RAHASIA PENULIS…]`, deterministik → cache prompt aman) agar Cek Konsistensi/Kontinuitas bisa menangkap kebocoran/kontradiksi; tetapi **disaring dari semua permukaan pembaca**: highlight editor & saran mention lewat `visibleCodexEntriesRef` (= `codexEntries.filter(!hidden)`) di `useEditorSetup.ts` (sengaja TERPISAH dari `codexEntriesRef` penuh yang tetap dipakai `ConsistencyUnderline`), dan ekspor `codexToMarkdown` (`src/lib/codexExport.ts` + tes) yang mengecualikan `hidden` (default, opsi `includeHidden`) dan **tak pernah** mencetak `secret`. UI: toggle+textarea di `CodexForm`, badge/blok di `CodexDetailModal` & `CodexCard`. **Ditunda ke fase 2:** penanda "reveal di bab N" + deteksi kebocoran deterministik. Lihat #1 di `RENCANA-FITUR-WORLDBUILDING.md`.

## Bengkel Nama (#3)
Field JSON `namePalette` di `CodexEntry`, logika murni `src/lib/nameForge.ts` (+ tes). Generator & glos nama patuh palet fonotaktik per-faksi; nol token, nol AI, deterministik (`generatePhonetic`/`composeMorphemic`/`glossName`, RNG `mulberry32` untuk tes ber-seed; `NamePalette`/`Morpheme` di `types.ts`). `namePalette` **inert**: BUKAN FK, tak diindeks, TAK masuk KB AI/ekspor Codex — ikut backup/impor otomatis sebagai bagian objek codex (jalur codex primer, tak di-remap). UI: `NameForgeModal` dibuka via tombol dadu di `CodexDetailModal` — alur berjenjang **preset (jalur utama) → generate → salin / "+ alias"**; kolom fonotaktik (onset/nukleus/coda dilabeli "bunyi awal/vokal/bunyi akhir") & leksikon morfem dilipat ke `<details>` "lanjutan". Chip preset punya state aktif (indigo + ✓) yang **luruh saat palet disunting manual**. Simpan palet ke `db.codex.update({namePalette})`. **Tombol "+ alias" di tiap nama hasil menulis ke `CodexEntry.aliases`** (baca-tulis dari DB agar tak clobber palet) — nama jadi langsung terlacak PresenceIndex/konteks. **Sengaja TIDAK dibangun (parkir, lihat #3):** flag ketaksesuaian fonotaktik — rawan false-positive, bertentangan dengan filosofi presisi Buku Gaya.

## Pelacak kelengkapan worldbuilding (#11, `viewMode 'completeness'`)
Dua field inert di `CodexEntry` + panel ringkas turunan (`WorldCompletenessPanel.tsx`), BUKAN tabel baru. `worldStatus?: 'stub'|'partial'|'solid'` (kematangan yang DIDEKLARASIKAN penulis) & `todo?: string` (catatan multi-baris, tiap baris = satu item). Skema Dexie **v27** no-op append-only. **Filosofi "penulis mendeklarasikan, alat membukukan":** bila `worldStatus` kosong, `effectiveStatus` jatuh ke SARAN lembut `suggestStatus` (heuristik kepadatan deskripsi + ada tidaknya field/secret) yang **tak disimpan** — badge menampilkannya dengan tanda "?". Logika murni `src/lib/worldCompleteness.ts` (+ tes): `suggestStatus`/`effectiveStatus`/`parseTodos`/`buildCompletenessReport`. **Deterministik & nol token — `worldStatus`/`todo` TIDAK masuk KB AI maupun `codexToMarkdown`** (murni metadata alur kerja; sengaja beda dari field #17). UI: segmen status + textarea TODO di `CodexForm` (persist di `useCodexPanel.handleSaveEntry` — `worldStatus` dibiarkan `undefined` bila "Otomatis"), badge `WorldStatusBadge` (`WorldStatusBadge.tsx`) di `CodexCard`/`CodexDetailModal` + blok TODO di modal, panel rekap bar bertumpuk. **Deep-link:** klik entri/TODO memakai `openCodexEntry(id)` di `NavigationContext` (`pendingCodexEntryId` → `CodexPanel` membuka `CodexDetailModal` via efek, lalu `clearPendingCodexEntry`) — primitif reusable membuka detail Codex dari panel lain.

## Integritas graf lore (#9)
Logika murni `src/lib/loreGraph.ts` (+ tes), NOL tabel/field baru. Graf DITURUNKAN dari data yang sudah ada — `buildLoreGraph(entries, relationships, promises)` mengembalikan `{ backlinks, dangling }`. **Backlink** dari tiga jalur: sebutan nama+alias lewat SATU scan Aho-Corasick atas `description`+`secret` (`'mention'`), relasi bertipe dua-arah (`'relationship'`), dan `PlotPromise.payoffCodexId` (`'payoff'`, `sourceId` diberi id janji NEGATIF agar tak bentrok id codex). **Tautan menggantung** = FK tersimpan yang menunjuk entri terhapus; `findDanglingRefs` menghitungnya SENDIRI tanpa membangun Aho-Corasick (dipakai banner). **UI (opsi C — audit tinggal di area Codex):** (1) seksi "Disebut oleh" di `CodexDetailModal` menampilkan backlink `via !== 'relationship'` (relasi dikecualikan — sudah dua-arah di seksi "Hubungan"), sebutan bisa diklik `onOpenEntry`; (2) `LoreIntegrityBanner` di header `CodexPanel` muncul HANYA bila ada yatim, resolusi lewat `resolveDangling` di `useCodexPanel` (relasi yatim → `db.relationships.delete`; FK janji yatim → `db.plotPromises.update(id,{[field]:undefined})`). Hapus satu entri sudah membersihkan relasinya (`confirmDelete`), TAPI TIDAK FK `plotPromises` → banner-lah jaring dangling janji. Deterministik & nol token.

## Graf visual (#14, `viewMode 'graph'`, `LoreGraphPanel.tsx`)
Menumpang `loreGraph.ts` yang sama: fungsi murni `buildLoreGraphView(entries, relationships, promises)` → `{ nodes, links }` (node per entri + `degree`; edge **tak-berarah** di-dedup dari tiga sumber — relasi/sebutan/payoff, tapi payoff kini `codexId→payoffCodexId` setup→pembayar, bukan id janji negatif; ujung menggantung disaring). Render pakai **`react-force-graph-2d`** (canvas + `d3-force`, code-split lazy). Node berwarna per-kategori lewat peta hex LOKAL di komponen (`CATEGORY_HEX`, karena `codexCategories.ts` hanya simpan kelas Tailwind); degree 0 diredam, entri `hidden` bercincin ungu putus-putus. Garis relasi diwarnai per-TIPE (`REL_TYPE_HEX`). Filter per-kategori & per-jenis-edge + "hanya terhubung" hanya mengubah state komponen (data tetap penuh). **Saat menambah sumber edge/analitik graf, perluas `buildLoreGraphView` + tesnya — jangan tambah tabel.**

> **⚠️ JEBAKAN BESAR (jangan diulang) — deteksi klik/hover ditangani SENDIRI lewat koordinat, BUKAN API bawaan force-graph.** Di kombinasi **`react-force-graph-2d` 1.29.1 + React 19 + render kanvas kustom**, hit-test bawaan lib hanya meregistrasi **SATU node** — sehingga `onNodeClick`, `onNodeHover`, dan tooltip `nodeLabel` cuma jalan untuk satu node. Gejala menyesatkan: "hanya satu node bisa diklik, dan kalau node itu dihapus, kemampuan-klik pindah ke node lain". BUKAN masalah ukuran area-klik / drag / `autoPauseRedraw`. **Solusi yang dipakai:** klik & tooltip dihitung manual dari `graph2ScreenCoords` — cari node terdekat ke titik kursor dalam radius layar (`nodeAtScreen`), di-wire di **wrapper `<div>`** (`onClick`/`onMouseMove`/`onMouseLeave`), plus tooltip DOM kustom (`hovered` state). `onNodeClick`/`onBackgroundClick`/`nodeLabel`/`nodePointerAreaPaint` bawaan **sengaja TIDAK dipakai** — mengembalikannya = regresi ke satu-node-saja. **Jangan "perbaiki" ini kembali ke API bawaan.** *(Panel Relasi (`RelationshipMapper`) sengaja pakai SVG radial kustom karena alasan sama.)*

- **Jebakan kecil:** ukuran kanvas diukur lewat **callback ref** (`setWrapEl`), bukan `useEffect`/`useLayoutEffect` — komponen `return null` selagi `useLiveQuery` memuat, jadi div kanvas belum ada saat effect ber-deps `[]` jalan → ref null → ResizeObserver tak terpasang → `size` selamanya 0×0 → `ForceGraph2D` tak ter-mount. Callback ref memasang observer tepat saat elemen menempel. Selain itu `base`/`graphData` di-memo lewat **signature string** (bukan identitas array) agar emit `useLiveQuery` identik tak me-restart simulasi.
- **Sorot-tetangga & layout:** hover node → node itu + tetangga langsung tetap terang, sisanya diredupkan (ego-network); dihitung dari `adjacency` + `hoverRef` yang dibaca `drawNode`/`linkColor` tiap frame. Ini butuh `autoPauseRedraw={false}` (redraw kontinu) — di sini SAH (untuk repaint sorot), BEDA dari upaya awal yang salah menyangka `autoPauseRedraw` penyebab bug klik. Node terisolasi didekatkan lewat gaya `d3Force('x'/'y', forceX/Y(0).strength(0.07))` — dipasang via ref tiap `graphData` berganti lalu `d3ReheatSimulation()`.

## Lokakarya Codex (`viewMode 'workshop'`)
Buat/edit entri Codex lewat diskusi AI. Panel dua kolom (kiri chat, kanan draf field terstruktur) di-route dari `MainView` TANPA `ViewContainer` (komponennya `absolute inset-0` sendiri). Dibuka via `openWorkshop(target)` di `NavigationContext` (state `workshopTarget: WorkshopTarget` + `closeWorkshop`); `'workshop'` **sengaja tak dipersist** ke localStorage → refresh kembali ke panel asal, tombol Kembali memakai `workshopReturnRef`. Otak di `hooks/useCodexWorkshop.ts`: chat pakai ulang `useChatSession`; **sesi dipersist** ke tabel `chatSessions` `kind:'workshop'` + field `workshopKey` (`entry:<id>` edit / `new:<seed>` create) untuk resume per-target — sesi **dibuang saat entri tersimpan** (lock modul-level anti double-create). **AI tak pernah menulis DB sendiri** — create=`db.codex.add`, edit=`db.codex.update` (mode edit tampilkan diff per-field + dialog before/after), keduanya `invalidateContextCache()` lalu `closeWorkshop()`. Titik masuk: tombol **Lokakarya** di header `CodexPanel` (create) & **Diskusikan** di `CodexDetailModal` (edit). Tiga jalur AI: (1) **live-fill** — AI menutup balasan dgn blok ```` ```codex-draft ```` (JSON), di-parse `parseCodexDraft`/disembunyikan `stripCodexDraft` (`src/lib/codexDraft.ts`, ada tes); instruksi dikirim lewat **`ChatParams.extraSystem`** (ditempel di akhir `systemInstruction`, TERPISAH dari KB agar cache `cachedContext` tetap byte-identik). (2) **harvest fallback** — tombol "Tarik dari diskusi" → `enrichEntities`. (3) **audit konsistensi** — `auditCodexEntry` (`services/ai/index.ts`, abort key `'codex-audit'`) dua lapis: **lapis 1** (`AUDIT_CODEX`, tombol "Audit") entri vs KB terstruktur (bukan teks bab); **lapis 2** (`AUDIT_CODEX_DEEP`, tombol "Mendalam", param `chapterExcerpts`) entri vs cuplikan prosa bab — dikumpulkan `gatherEntityContext` (nama+alias, cap 4k char) dari bab ter-strip HTML. Temuan ditempel lewat `appendAuditMessage` (ikut dipersist).
